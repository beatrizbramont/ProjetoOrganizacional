Site ou aplicativo que fará/terá:
- calendário
- "bloco de notas"
- Despertador

🎯 1- Definir escopo mínimo (MVP)
Para não se perder e começar de forma enxuta, você deve ter um produto mínimo funcional com:

1️⃣ Cadastro/login de usuários
2️⃣ Calendário para adicionar eventos
3️⃣ Bloco de notas
4️⃣ Agendamento de lembretes com notificação (push ou e-mail)
5️⃣ Interface responsiva no celular
6- dar a oportunidade ao usuário escolher personalizar o app com o seu jeito

🌐 4 - Criar rotas e APIs
/register (cadastro)
/login /logout
/notes (listar, criar, editar, excluir notas)
/events (listar, criar, editar, excluir eventos)
/reminders (agendar lembrete)
/api/ endpoints REST (caso queira conectar a um frontend mobile depois)

🖥️ 5- Criar interface frontend responsiva
Sugestão de stack:
HTML5 + Bootstrap (ou Tailwind)
Calendário: FullCalendar.js
Bloco de notas: textarea simples ou editor rico (Quill.js)
PWA manifest.json + service workers (permite instalar no celular)

🕑 7- Tarefas agendadas
Para disparar lembretes no horário certo:
✅ Use Celery + Redis para agendar tarefas em background
Ex.:
O usuário agenda lembrete para 14h
Celery espera até 14h e dispara notificação
